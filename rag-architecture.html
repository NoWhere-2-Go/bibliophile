Phase I · Offline Ingestion Pipeline

SOURCE 📚
Book Corpus
PDFs, EPUBs, plain text. Project Gutenberg, curated library, or user uploads.
parse
PROCESS ✂️
Text Chunker
Recursive character splitting. ~512 token chunks with 64 token overlap. Metadata preserved: title, author, genre, chapter.
embed
MODEL 🔢
Embedding Model
nomic-embed-text (Ollama) or sentence-transformers. Produces dense vectors per chunk.
index
🗄️
Vector Store
ChromaDB or Qdrant. Stores embeddings + metadata. Persists to disk between runs.
ChromaDB or Qdrant

Phase II · Runtime Query Pipeline

INPUT 💬
User Query
Natural language request. e.g. "Books like Dune with philosophical depth" or genre/mood filters.
embed
PROCESS 🧭
Query Embedding
Same embedding model as ingestion. Query → dense vector for similarity search.
top-k ANN
RETRIEVE 🔍
Semantic Retriever
Cosine similarity search. Top-k=5–10 chunks returned. Optional metadata filter (genre, era, rating).
augment
BUILD 📝
Context Builder
Assembles prompt with retrieved passages + user query. Formats system prompt for recommendation style.

Phase III · Generation & Response

PROMPT 📜
Augmented Prompt
System role + retrieved context chunks + user query. Structured to elicit ranked recommendations with rationale.
infer
LLM 🧠
Reasoning Model
Local inference via Ollama API (localhost:11434) or DeepSeek R1 API. Reads context, reasons over book passages, generates nuanced recommendations.
DeepSeek R1 API · deepseek-r1:7b Ollama · deepseek-r1:14b Ollama
respond
OUTPUT ✨
Recommendation
Ranked list with summaries, similarity rationale, and passage excerpts. Streamed to UI.

🛠 Stack

  • Python 3.11+ backend
  • LangChain or LlamaIndex for orchestration
  • FastAPI for REST / WebSocket
  • Ollama running locally
  • ChromaDB persisted to ./chroma_db

📊 Embedding Strategy

  • nomic-embed-text (768-dim, via Ollama)
  • Chunk size 512 tokens, overlap 64
  • Metadata: title, author, genre, year
  • Re-ranking optional (cross-encoder)

🔀 Model Routing

  • Env var selects DeepSeek API vs Ollama
  • Ollama: deepseek-r1:7b (fast) or :14b (quality)
  • Streaming responses via SSE
  • Fallback chain on model unavailability

⚡ Performance

  • Ingestion is one-time / incremental
  • Query latency dominated by LLM inference
  • Cache embeddings; skip re-ingested books
  • GPU acceleration via Ollama if available

🔒 Offline-First

  • Full pipeline runs air-gapped with Ollama
  • No data sent externally (unless DeepSeek API)
  • Books stay on local filesystem
  • ChromaDB persists between restarts

🚀 Extensions

  • Hybrid search (BM25 + vector) for recall
  • User preference memory across sessions
  • Goodreads / OpenLibrary metadata enrichment
  • Reranker: ms-marco-MiniLM cross-encoder
